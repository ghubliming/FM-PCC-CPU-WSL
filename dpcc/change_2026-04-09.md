# Change Log — 2026-04-09

## Fix: `NotImplementedError` in `sequence_dataset` for `avoiding-d3il-visual`

### Problem

Training the visual avoiding task via:

```bash
python FM_v3_avoiding_visual_test/train_FM_v3_avoiding_visual.py --seeds 6
```

crashed with:

```
File "flow_matcher_v3_avoiding_visual/datasets/d4rl.py", line 162, in sequence_dataset
    raise NotImplementedError
NotImplementedError
```

The `sequence_dataset()` function in `d4rl.py` only matched env names `'avoiding-d3il'` and `'d3il-avoiding'`, but the visual training script passes `env='avoiding-d3il-visual'` (set in `train_FM_v3_avoiding_visual.py` line 13). This unrecognised name fell through to the `else: raise NotImplementedError` branch.

### Root Cause

The visual variant uses the **same raw d3il pickle data** as the non-visual variant — the "visual" distinction lives in the model architecture, not the data source. The env name simply wasn't listed in the condition.

### Fix

**File:** `flow_matcher_v3_avoiding_visual/datasets/d4rl.py` (line 136)

```diff
- elif env == 'avoiding-d3il' or env == 'd3il-avoiding':
+ elif env in ('avoiding-d3il', 'd3il-avoiding', 'avoiding-d3il-visual'):
```

This allows the visual variant to reuse the existing d3il avoiding data loader without any other changes.
